# 门禁规则与流转校验

> 从 RULES.md 提取。PMO 每次阶段变更必须遵循此规范。

## 🔴 PMO 阶段门禁规则（Phase Gate）

**PMO 在启动任何阶段前，必须执行以下门禁检查，禁止跳过：**

```
PMO 启动阶段 X 前：
├── 1. 查阅「阶段状态转移表」→ 确认当前阶段到阶段 X 的路径合法
├── 2. 验证前置阶段产物是否存在：
│   ├── PRD 确认后才能进 QA/Designer
│   ├── TC 评审通过后才能进 RD 技术方案
│   ├── 技术方案确认后才能进 RD 开发
│   └── 架构师 Code Review 通过后才能进 QA 代码审查
├── 3. 验证暂停点是否已获用户确认（检查对话历史中是否有用户明确回复）
└── 4. 如果门禁不通过 → 🔴 禁止进入下一阶段，输出缺失项提醒用户

违规示例（禁止）：
├── ❌ TC 评审通过后直接进入 RD 开发（跳过了技术方案 + 架构师 Review）
├── ❌ PRD 确认后直接进入 RD 开发（跳过了 QA TC + 技术方案）
└── ❌ 技术方案未经用户确认就进入开发

PMO 不能以「方案简单」「时间紧迫」「用户没提到」为由自行跳过任何阶段。
跳过阶段的唯一合法路径：RD 提出跳过申请 → ⏸️ 用户明确同意。
```

## 🔴 强制流转校验输出（每次阶段变更必须输出）

**PMO 每次推进到下一阶段时，必须在切换前输出以下校验块。缺少校验块 = 非法流转。**

```
📋 流转校验
├── 流程类型：{Feature / Bug 处理 / 问题排查 / Feature Planning / 敏捷需求}
├── 当前阶段：{规范阶段名}
├── 目标阶段：{规范阶段名}
├── 转移表匹配：✅ 合法 / ❌ 非法（附转移表行号或原文）
├── STATUS.md 禁止列表检查：✅ 目标不在禁止列表中 / ❌ 目标在禁止列表中 → 🔴 阻塞
├── 前置条件：✅ {条件} 已满足 / ❌ {条件} 未满足
├── 暂停点检查：
│   ├── 目标阶段前是否有暂停点：是 / 否
│   └── 用户是否已确认：✅ 已确认（引用用户原文） / ❌ 未确认 → 🔴 阻塞
└── 校验结果：✅ 允许流转 / ❌ 阻塞（原因）
```

### 🔴 执行规则

```
├── 校验结果为 ✅ → 继续流转，校验块作为 PMO 阶段摘要的一部分输出
├── 校验结果为 ❌ → 禁止流转，输出阻塞原因并等待问题解决
├── 缺少校验块直接切换阶段 → 违反红线第 5 条
└── 以下场景免输出校验块：
    ├── 同一阶段内的角色操作（如 RD 开发中的子步骤）
    └── Subagent 内部的步骤推进（Subagent 不感知 PMO 校验）
```

## 🔴 STATUS.md 流转约束同步更新

**PMO 每次完成流转校验并允许流转后，必须同步更新 STATUS.md 的「流转约束」段：**

```
更新内容（从阶段状态转移表查出）：
├── 当前流程 → 保持不变（除非流程切换，如问题排查 → Feature）
├── 当前阶段 → 更新为刚进入的新阶段
├── 合法下一阶段 → 查转移表，填写新阶段允许的后续阶段
├── 禁止跳转到 → 列出当前阶段之后的所有非相邻阶段（🔴 新增）
├── 流转条件 → 填写后续阶段的触发条件
├── 是否暂停点 → 查转移表，判断当前新阶段完成后是否有 ⏸️
├── 当前待确认项 → 暂停点时填写具体待确认内容（如「用户确认 PRD」「选择 A/B/C 方案」），非暂停点填「无」
└── 回退路径 → 如有 🔁 则填写，否则填「无」

示例（Feature 流程，刚进入 QA Write Cases 阶段）：
├── 当前阶段：QA Write Cases
├── 合法下一阶段：TC 评审
├── 禁止跳转到：RD 技术方案 / 架构师 Review / RD 开发 / RD 自查 / 架构师 Code Review / QA 代码审查 / PM 验收
├── 流转条件：三类 Case 完成（BDD + API E2E + Browser E2E）
├── 是否暂停点：否（TC 评审自动启动 Subagent）
├── 当前待确认项：无
└── 回退路径：无
```

### 🔴 执行要求

```
├── 每次阶段流转后必须更新，缺失 = 违规
├── 更新时机：流转校验通过 → 更新 STATUS.md 流转约束 → 切换角色开始新阶段
├── 🔴「禁止跳转到」是硬约束：PMO 流转时如果目标阶段在禁止列表中 → 阻塞 + 报错
├── compact 恢复时，PMO 只需读 STATUS.md 即可获得当前流转约束，无需重新查转移表
└── 如果 STATUS.md 流转约束段缺失或为空 → PMO 必须从转移表重建后再继续
```
